编程练习2：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 $R^2$ 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

$R^2$ 的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的 $R^2$ 值为0还不如直接用平均值来预测效果好；而一个 $R^2$ 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的 $R^2$，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

  • 将他们的表现评分储存到 score 变量中。

如果用模型已经见过的数据，例如部分训练集数据进行测试，又有什么坏处？

问题 3 - 回答:¶

将数据拆分成训练集和测试集是为了得到泛化能力高的模型。用训练集的数据来训练，得到模型之后，再用测试集（相当于新的数据）来测试模型，观察模型对未接触过的数据是否仍能表现良好，这样避免得到的模型过拟合，无法运用来解决新的问题。

如果用于测试的数据是模型已经见过的，那么其实就起不到测试的作用了，仍然有可能是模型对已知数据区分良好，而未能检测对未知数据的区分能力。

问题 5 - 回答:¶

当模型以最大深度1训练时，预测出现很大的偏差（bias），而模型以最大深度10训练时，出现了很大的方差（variance）。

因为训练深度1是一个简单的模型，很有可能欠拟合。深度10是较复杂的模型，相比于1，更有可能出现过拟合。在图中表现为，深度为1时评分低，即因变量的变化中的40%左右可以由自变量解释，是不足的。

随着训练深度的增加，训练集的R^2越来越大，自变量能解释的变化越多，说明模型在训练集中拟合得越来越好。

但是在验证集中的表现并不理想，最大深度1~3时，R^2随着深度的增大而越来越高，模型表现越来越好，但在深度更大时却出现了下降，与训练集曲线距离越来越大，说明该模型在验证数据上表现并不好，因变量的变化不再能充分由自变量解释，说明在训练集上出现了过拟合。

  1. K折交叉验证法是将数据分为K个大小近似的互斥子集，模型训练K次，每次将其中的一个子集作为测试集，每次所选的子集都不重复；其余k-1个子集作为训练集。此过程中还可将数据进行随机化，以消除偏倚。训练并验证K次之后得到K个结果（如准确率、偏差等等），将K次的结果平均做为其估计值，得到最终的单一模型。
  1. GridSearchCV()可以将网格搜索法与K折交叉验证相结合。在网格搜索的策略下，我们先建立各超参数的所有可能的组合，建立这些组合的网格；结合K折交叉验证法的话，则每套超参数组合的模型，又按K折交叉验证法进行K次训练并验证，得到K个评价分数，将它们平均，做为该套超参数组合的分数；各套超参数组合的分数再参与网格搜索体系的比较，也就是遍历各套组合的结果之后，选出最佳模型。GridSearchCV()中的cv参数可以设置交叉验证策略。0.22之后的版本是默认5折交叉验证，可以改为其他整数。也可以传入训练和测试数据的下标的矩阵等其他的分割器。
  1. cv_results_属性会以字典（包装成表格）的形式罗列出交叉验证的超参数组合、各种评分等信息。字典的键（列名）即超参数、评分体系等名称，字典的值（各列值）即各超参数的取值以及各评分的结果。还有一列params以多个字典的列表的形式储存所有超参数组合。
  1. 网格索索中的每套参数组合，都经过不同数据的训练和验证并进行综合，可以使模型更稳定。K折交叉验证也可以避免损失一部分训练数据，充分利用所有数据进行训练。如果有将数据随机化，还可能避免偏倚。

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(n_splits=10, test_size=0.20, random_state=42)
    
    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': range(10)[-9:]}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(r2_score)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor, params, scoring = scoring_fnc, cv = cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
